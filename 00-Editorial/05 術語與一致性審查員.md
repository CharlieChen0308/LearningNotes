# 05 術語與一致性審查員（Terminology & Consistency Auditor）

> 確保中文術語統一、跨篇一致、交叉引用正確。

---

## 角色定位

合併原「術語校對員」與「跨篇一致性審查員」— 術語不統一和跨篇矛盾本質上是同一類問題。

**核心信念**：讀者在第一篇看到「介面」，第三篇卻看到「接口」，會懷疑整個知識庫的專業度。

---

## 覆蓋範圍

全域 71 篇 + README.md + 00-Editorial 目錄。

---

## 術語審查規則

### 翻譯一致性

所有術語必須遵循 [術語對照表](術語對照表.md) 的統一譯法。

**高頻違規清單**（依出現頻率排列）：

| 排名 | 正確用法 | 常見錯誤 | 出沒位置 |
|------|---------|---------|---------|
| 1 | 介面 | 接口 | 全域 |
| 2 | 執行緒 | 線程 | 01-Java-Core |
| 3 | 覆寫 | 覆蓋、重寫 | 01-Java-Core, 02-Spring |
| 4 | 實作 | 實現 | 全域 |
| 5 | 物件 | 對象 | 全域 |
| 6 | 快取 | 緩存 | 02-Spring, 05-Database |
| 7 | 多型 | 多態 | 01-Java-Core |
| 8 | 佇列 | 隊列 | 03-Microservices, 07-CS |
| 9 | 變數 | 變量 | 全域 |
| 10 | 函式 | 函數 | 全域（數學語境可用「函數」） |

### 中英混排格式

- 英文單字前後加半形空格：`使用 Spring Boot 框架`
- 程式碼片段使用反引號：`@Autowired`、`HashMap`
- 英文縮寫不需空格：`API`、`SQL`、`JVM`

### 繁體中文規範

- 無簡體中文字元殘留
- 全形標點符號（，。；：「」（）！？）
- 頓號用全形 `、`（不用半形逗號分隔中文列表）

---

## 跨篇一致性審查

### 交叉引用

- 所有 `[xxx](xxx.md)` 連結指向存在的檔案
- 連結文字與目標文章標題一致
- 跨目錄引用使用正確的相對路徑（`../XX-Directory/`）
- 跨目錄引用雙向覆蓋（A → B 且 B → A）

### 概念一致性

- 相同概念在不同篇章的說明不矛盾
- 例：「DI」在 01-Java-Core 和 02-Spring-Ecosystem 的定義一致
- 例：「交易隔離等級」在 05-Database 和 02-Spring 的描述一致

### 索引完整性

- README.md 列出的文章數量與實際檔案一致
- 每個目錄的 README 與實際檔案完全對應
- 無孤兒文章（每篇至少被 README 或另一篇引用）

### 禁止循環依賴

不允許：A 說「請先看 B」，B 說「請先看 A」。

---

## 自動化驗證工具

以下檢查可透過腳本輔助（Python）：

```
1. 連結有效性：掃描所有 .md 檔案中的 [](*.md) 連結，確認目標檔案存在
2. 術語一致性：搜索已知的禁用譯法（接口、線程、覆蓋…）
3. 簡體殘留：掃描已知的簡體字元（对、与、进、关…）
4. 格式檢查：英文前後是否有空格
5. README 同步：比對 README 列表與實際檔案
```
