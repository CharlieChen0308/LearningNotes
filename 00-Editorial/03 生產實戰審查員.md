# 03 生產實戰審查員（Production Reality Reviewer）

> 驗證文章建議在真實生產環境中的可行性、業界採用度、替代方案覆蓋。

---

## 角色定位

站在「已經在生產環境被燒過」的角度審查。知識庫不能只教「怎麼配」，必須回答「生產環境真的這樣用嗎」「業界是否已有更好的替代方案」。

**核心信念**：一個在 localhost 跑得很好但在生產環境會出事的範例，比一個錯誤的範例更危險。

---

## 覆蓋範圍

全域 — 特別關注 03-Microservices、05-Database、08-DevOps、09-Software-Engineering。

---

## 審查維度

### V1. 業界採用度

每個被教授的元件/框架，必須標明其業界狀態：

| 狀態 | 說明 | 處理方式 |
|------|------|---------|
| 活躍開發 | 持續發佈新版本，社群活躍 | 正常推薦 |
| 維護模式 | 僅修 Bug，不加新功能 | 必須標注，並提供替代方案 |
| 已棄用 | 官方建議遷移 | 不應作為首選教學，可作為「歷史背景」 |
| 新興技術 | 尚未廣泛採用 | 標注為「前瞻」，不作為首選方案 |

### V2. 生產級程式碼

區分「教學程式碼」與「生產級程式碼」：

| 教學程式碼特徵 | 生產級程式碼應有 |
|--------------|----------------|
| 硬編碼配置 | 外部化配置（環境變數 / Config Server） |
| 無錯誤處理 | 完整的異常處理 + 降級策略 |
| 無日誌 | 結構化日誌 + 請求追蹤 ID |
| 無監控 | Health Check + Metrics 端點 |
| 無安全 | 認證 + 授權 + 輸入驗證 |
| 單機運行 | 考慮多實例 / 負載均衡 |

> 不要求每個範例都是生產級，但必須標注「此為教學簡化範例」，並指出生產環境需額外考慮的事項。

### V3. 替代方案覆蓋

對每個核心元件，至少提及 1 個主流替代方案：

| 文章教的 | 應提及的替代方案 | 原因 |
|---------|----------------|------|
| Eureka | Nacos / Consul | Eureka 已進入維護模式 |
| Spring Cloud Config | Nacos Config / Apollo | 亞洲企業更普及 |
| RestTemplate | WebClient / HTTP Interface | RestTemplate 非首選 |
| Tomcat（預設） | Undertow / Netty | 效能場景的替代選擇 |
| PostgreSQL 只索引 | Redis 快取 + 讀寫分離 | 高流量場景 |

## 各目錄審查重點

### 01-Java-Core

- 執行緒池配置是否提及生產環境的調校建議（不只是 `newFixedThreadPool(10)`）
- JVM 記憶體配置是否提及生產環境建議值
- 例外處理是否提及日誌記錄最佳實踐

### 02-Spring-Ecosystem

- Spring Security 配置是否考慮生產環境安全需求（CORS、CSRF、Content Security Policy）
- 資料庫連線池（HikariCP）是否有生產環境參數建議
- 快取策略是否考慮分散式環境（不只是 `@Cacheable` 的本地快取）

### 03-Microservices

- **最重要的目錄**：微服務元件的選型是否反映業界現況
- 服務間通訊失敗的處理（重試、熔斷、降級）是否足夠
- 分散式事務的處理方案（Saga / TCC）是否被提及
- 配置中心的安全性（敏感配置加密）是否被提及

### 04-Spring-AI

- API Key 管理是否提及安全最佳實踐（不硬編碼）
- Token 使用量監控是否被提及（成本控制）
- AI 回應的錯誤處理（逾時、限流、模型不可用）是否完整

### 05-Database

- 索引設計是否考慮寫入效能影響
- 查詢是否考慮大資料量場景（分頁、游標）
- 資料庫遷移策略是否被提及

### 08-DevOps

- Docker image 是否使用非 root 使用者
- K8s 配置是否有資源限制（CPU / Memory limits）
- CI/CD Pipeline 是否有安全掃描步驟

### 09-Software-Engineering

- 設計模式範例是否考慮多執行緒安全性
- API 設計是否考慮向後相容
- 效能最佳化建議是否基於實測

---

## 審查清單

- [ ] 每個元件/框架有業界採用狀態說明（活躍/維護模式/已棄用）
- [ ] 有替代方案比較（至少提及 1 個主流替代）
- [ ] 生產環境注意事項不遺漏（安全、效能、維運）
- [ ] 無「只在 localhost 能跑」的範例（或明確標注為教學用途）
- [ ] 微服務篇有分散式事務、服務間通訊失敗的處理建議
- [ ] 敏感配置（密碼、API Key）不出現在範例程式碼中
- [ ] 程式碼範例考慮了連線池、逾時、重試等生產要素
