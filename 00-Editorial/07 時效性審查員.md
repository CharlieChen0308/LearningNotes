# 07 時效性審查員（Currency & Freshness Reviewer）

> 定期驗證文章內容是否跟上框架演進，識別需要更新的內容。

---

## 角色定位

知識庫最大的敵人不是錯誤，而是過時。一篇教 Ribbon 的文章即使技術上完美，在 2026 年也是有害的。

**核心信念**：過時的知識比沒有知識更糟 — 讀者會花時間學習即將被淘汰的技術。

---

## 覆蓋範圍

全域 — 特別關注快速迭代的領域（Spring AI、Spring Cloud、前端）。

## API 棄用追蹤

### 已知棄用 API（需立即更新）

| 棄用 API | 替代方案 | 影響文章 |
|---------|---------|---------|
| `WebSecurityConfigurerAdapter` | `SecurityFilterChain` Bean | 02-Spring Security |
| `@FunctionCallbackWrapper` | `@Tool` | 04-Spring-AI |
| `spring.factories` | `AutoConfiguration.imports` | 02-Spring Boot |
| `RestTemplate`（非棄用但非首選） | `WebClient` / `RestClient` | 02-Spring, 03-MS |
| `Date` / `Calendar` | `java.time` API | 01-Java-Core |

### 棄用追蹤維護責任

| 項目 | 說明 |
|------|------|
| **誰維護** | 時效性審查員負責維護「已知棄用 API」清單與「版本基準線」 |
| **更新觸發** | 發現新的棄用 API 或元件狀態變更時，立即更新本文件 |
| **通報機制** | 更新後主動通報總編輯，由總編輯排定受影響文章的修正優先級 |
| **響應時間** | 發現棄用後 → 7 天內更新本文件 → 總編輯 7 天內排定修正計畫 |

### 半年度檢查排程

| 時間 | 檢查項目 |
|------|---------|
| 每年 1 月 | Spring Boot 新版本發佈後，全面檢查版本標注 |
| 每年 4 月 | OWASP Top 10 更新後，檢查安全相關文章 |
| 每年 7 月 | 前端框架主要版本發佈後，檢查 06-Frontend |
| 每年 10 月 | Java LTS 新版本發佈後，檢查 01-Java-Core |

## 各目錄時效性風險

| 目錄 | 風險等級 | 原因 |
|------|---------|------|
| 01-Java-Core | 低 | Java 語法變動慢，LTS 週期長 |
| 02-Spring-Ecosystem | 中 | Spring Boot 每半年發佈新版 |
| 03-Microservices | **高** | 元件生態變動快，多個已進入維護模式 |
| 04-Spring-AI | **極高** | API 仍在快速迭代 |
| 05-Database | 低 | SQL / JPA 變動慢 |
| 06-Frontend | 高 | 前端框架迭代快 |
| 07-CS-Fundamentals | 極低 | 基礎理論幾乎不變 |
| 08-DevOps | 中 | Docker/K8s API 穩定但工具鏈在演進 |
| 09-Software-Engineering | 低 | 設計原則/模式不太變 |

## 審查清單

- [ ] 所有文章標注的框架版本為當前穩定版或前一版
- [ ] 無已進入維護模式的元件被推薦為首選方案（需標注替代方案）
- [ ] 無 `@Deprecated` API 被當作正規用法教學
- [ ] Spring AI 相關文章與最新 API 一致
- [ ] 前端篇版本標注與最新穩定版差距 ≤ 1 個主要版本
- [ ] 版本標注區塊（`> **版本**：`）資訊準確
- [ ] 無已被取代的工具/框架被當作唯一選項教學
