# 02 查詢演算法總覽

> **版本**：Java 17+ — 所有程式碼基於 Java 17 編寫

查詢（Search）是電腦科學中最基礎的操作之一。從簡單的陣列遍歷到雜湊表的常數級查詢，不同的演算法適用於不同的資料特性與使用場景。本文將五種經典查詢演算法整合為一篇，涵蓋原理、Java 實作與標準庫對照。

---

## 1、查詢演算法比較表

| 演算法 | 時間複雜度（平均） | 時間複雜度（最差） | 空間 | 前提條件 |
|--------|-------------------|-------------------|------|---------|
| 順序查詢 | O(n) | O(n) | O(1) | 無 |
| 二分查詢 | O(log n) | O(log n) | O(1) | 已排序的順序表 |
| 索引查詢 | O(sqrt(n)) | O(n) | O(sqrt(n)) | 分塊有序 + 索引表 |
| 二叉排序樹查詢 | O(log n) | O(n) | O(n) | 建構 BST |
| 雜湊查詢 | O(1) | O(n) | O(n) | 雜湊函式 + 處理衝突 |

> **提示**：「平均」複雜度假設資料分佈均勻且結構平衡。最差情況通常出現在極端資料分佈下（如有序插入 BST 導致退化為鏈表）。

## 2、順序查詢（Sequential Search）

順序查詢是最直覺的查詢方式：從頭到尾逐一比對，找到即返回，遍歷完畢未找到則回傳 -1。

**特性**：
- 時間複雜度 O(n)，無需任何前置條件
- 適合小型資料集或未排序的資料
- 可同時用於陣列與鏈結串列

```java
public class SequentialSearch {

    /**
     * 順序查詢：逐一比對，找到回傳索引，否則回傳 -1
     */
    public static int search(int[] arr, int key) {
        for (int i = 0; i < arr.length; i++) {
            if (arr[i] == key) {
                return i;
            }
        }
        return -1;
    }

    public static void main(String[] args) {
        int[] data = {90, 10, 20, 50, 70, 40, 80, 60, 30, 52};
        int index = search(data, 50);
        System.out.println(index != -1
                ? "找到 50，位於索引 " + index
                : "未找到");
    }
}
```

**輸出**：`找到 50，位於索引 3`

## 3、二分查詢（Binary Search）

二分查詢的核心思想是「每次排除一半」：在已排序的陣列中，取中間元素與目標比較，根據大小關係縮小搜尋範圍。

**前提條件**：
1. 資料必須**已排序**
2. 必須使用支援隨機存取的結構（如陣列）

### 3-1、迭代版本

```java
public class BinarySearchIterative {

    public static int search(int[] arr, int key) {
        int low = 0;
        int high = arr.length - 1;

        while (low <= high) {
            // 避免 (low + high) 溢位的寫法
            int mid = low + (high - low) / 2;

            if (arr[mid] == key) {
                return mid;
            } else if (arr[mid] > key) {
                high = mid - 1;
            } else {
                low = mid + 1;
            }
        }
        return -1;
    }

    public static void main(String[] args) {
        int[] data = {10, 20, 30, 40, 50, 60, 70, 80, 90};
        int index = search(data, 30);
        System.out.println(index != -1
                ? "找到 30，位於索引 " + index
                : "未找到");
    }
}
```

### 3-2、遞迴版本

```java
public class BinarySearchRecursive {

    public static int search(int[] arr, int key, int low, int high) {
        if (low > high) {
            return -1;
        }
        int mid = low + (high - low) / 2;

        if (arr[mid] == key) {
            return mid;
        } else if (arr[mid] > key) {
            return search(arr, key, low, mid - 1);
        } else {
            return search(arr, key, mid + 1, high);
        }
    }

    public static void main(String[] args) {
        int[] data = {10, 20, 30, 40, 50, 60, 70, 80, 90};
        int index = search(data, 50, 0, data.length - 1);
        System.out.println(index != -1
                ? "找到 50，位於索引 " + index
                : "未找到");
    }
}
```

### 3-3、常見陷阱

**中間值溢位問題**：當 `low` 和 `high` 都接近 `Integer.MAX_VALUE` 時，`(low + high)` 會發生整數溢位。正確做法是使用 `low + (high - low) / 2`，或者使用無符號右移 `(low + high) >>> 1`。

```java
// 錯誤寫法：當 low + high 超過 Integer.MAX_VALUE 時溢位
int mid = (low + high) / 2;

// 正確寫法 A：減法避免溢位
int mid = low + (high - low) / 2;

// 正確寫法 B：無符號右移（Java 標準庫 Arrays.binarySearch 的做法）
int mid = (low + high) >>> 1;
```

## 4、索引查詢（Index Search）

索引查詢又稱**分塊查詢**，是介於順序查詢與二分查詢之間的方法。基本思想是：

1. 將主表分成若干**塊**，每塊內部不要求有序，但塊與塊之間必須有序（前一塊的最大值 < 後一塊的最小值）
2. 建立**索引表**，每個索引項記錄該塊的最大關鍵字與起始位置
3. 查詢時先在索引表中定位目標所在的塊（可用二分查詢），再在塊內做順序查詢

**索引項結構**：每個索引項包含三個資訊 — 塊的最大關鍵字（maxKey）、塊在主表中的起始位置（start）、塊的長度（length）。

```java
public class IndexSearch {

    record IndexItem(int maxKey, int start, int length) {}

    /**
     * 索引查詢
     *
     * @param mainTable  主表
     * @param indexTable 索引表
     * @param key        目標值
     * @return 目標值在主表中的索引，未找到回傳 -1
     */
    public static int search(int[] mainTable, IndexItem[] indexTable, int key) {
        // 第一步：在索引表中找到目標所在的塊
        IndexItem targetBlock = null;
        for (IndexItem item : indexTable) {
            if (key <= item.maxKey()) {
                targetBlock = item;
                break;
            }
        }
        if (targetBlock == null) {
            return -1;
        }

        // 第二步：在塊內順序查詢
        int start = targetBlock.start();
        int end = start + targetBlock.length();
        for (int i = start; i < end; i++) {
            if (mainTable[i] == key) {
                return i;
            }
        }
        return -1;
    }

    public static void main(String[] args) {
        // 主表：三個塊，塊間有序，塊內不要求有序
        int[] mainTable = {8, 5, 2, 10, 18, 12, 15, 20, 25, 30, 22, 28};

        // 索引表
        IndexItem[] indexTable = {
                new IndexItem(10, 0, 4),   // 第一塊：最大值 10，從 0 開始，長度 4
                new IndexItem(20, 4, 4),   // 第二塊：最大值 20，從 4 開始，長度 4
                new IndexItem(30, 8, 4)    // 第三塊：最大值 30，從 8 開始，長度 4
        };

        int index = search(mainTable, indexTable, 22);
        System.out.println(index != -1
                ? "找到 22，位於索引 " + index
                : "未找到");
    }
}
```

**輸出**：`找到 22，位於索引 10`

**複雜度分析**：若主表長度為 n，分成 b 塊，每塊長度約 s = n/b。索引表查詢 O(b)（順序）或 O(log b)（二分），塊內查詢 O(s)。當 b = sqrt(n) 時，平均查詢長度最優，約 O(sqrt(n))。

## 5、二叉排序樹查詢（BST Search）

二叉排序樹（Binary Search Tree, BST）具備以下性質：

- 左子樹所有節點的值 **小於** 根節點
- 右子樹所有節點的值 **大於** 根節點
- 左右子樹也各自是二叉排序樹

查詢時從根節點開始，目標比節點小就往左走，比節點大就往右走，相等則找到。

```java
public class BSTSearch {

    static class Node {
        int data;
        Node left, right;

        Node(int data) {
            this.data = data;
        }
    }

    /** 插入節點 */
    public static Node insert(Node root, int value) {
        if (root == null) {
            return new Node(value);
        }
        if (value < root.data) {
            root.left = insert(root.left, value);
        } else if (value > root.data) {
            root.right = insert(root.right, value);
        }
        // 若相等則不重複插入
        return root;
    }

    /** 查詢節點 */
    public static boolean search(Node root, int key) {
        if (root == null) {
            return false;
        }
        if (key == root.data) {
            return true;
        }
        return key < root.data
                ? search(root.left, key)
                : search(root.right, key);
    }

    /** 中序遍歷（結果為升序） */
    public static void inOrder(Node root) {
        if (root != null) {
            inOrder(root.left);
            System.out.print(root.data + " ");
            inOrder(root.right);
        }
    }

    public static void main(String[] args) {
        int[] values = {50, 30, 70, 10, 40, 90, 80};
        Node root = null;
        for (int v : values) {
            root = insert(root, v);
        }

        System.out.print("中序遍歷：");
        inOrder(root);
        System.out.println();

        System.out.println("查詢 40：" + search(root, 40));
        System.out.println("查詢 25：" + search(root, 25));
    }
}
```

**輸出**：

```
中序遍歷：10 30 40 50 70 80 90
查詢 40：true
查詢 25：false
```

### 退化問題與平衡樹

當資料以**遞增或遞減順序**插入時，BST 會退化成一條鏈表，查詢時間退回 O(n)。解決方案是使用**自平衡二叉搜尋樹**：

- **AVL 樹**：嚴格平衡，任何節點的左右子樹高度差不超過 1，查詢穩定 O(log n)
- **紅黑樹（Red-Black Tree）**：近似平衡，插入/刪除效率優於 AVL

Java 標準庫中的 `TreeMap` 與 `TreeSet` 內部就是基於**紅黑樹**實作的，保證所有操作的時間複雜度為 O(log n)。

## 6、雜湊查詢（Hash Search）

雜湊查詢的核心概念是透過**雜湊函式**將 key 直接映射到儲存位置，理想情況下可達到 O(1) 的查詢效率。

### 6-1、雜湊函式

常見的雜湊函式構造方法：

- **除留餘數法**（最常用）：`f(key) = key % tableSize`
- **直接定址法**：`f(key) = a * key + b`
- **平方取中法**：取 key 平方後的中間幾位數
- **摺疊法**：將 key 分段後疊加求和

### 6-2、衝突處理

當兩個不同的 key 映射到同一位置時（即 `f(key1) == f(key2)` 但 `key1 != key2`），稱為**衝突**。兩種主要的解決策略：

**鏈結法（Chaining）**：每個雜湊桶維護一個鏈結串列，衝突的元素接在同一串列上。Java 的 `HashMap` 在 JDK 8 以後採用此法，當鏈結長度超過 8 時會轉換為紅黑樹以提升效能。

**開放定址法（Open Addressing）**：衝突時依照探測序列尋找下一個空位。常見策略有線性探測、二次探測、雙重雜湊等。

### 6-3、Java 實作（鏈結法）

以下示範一個簡化版的雜湊表，使用鏈結法處理衝突：

```java
import java.util.LinkedList;

public class SimpleHashTable {

    private static final int TABLE_SIZE = 7;
    private final LinkedList<Integer>[] buckets;

    @SuppressWarnings("unchecked")
    public SimpleHashTable() {
        buckets = new LinkedList[TABLE_SIZE];
        for (int i = 0; i < TABLE_SIZE; i++) {
            buckets[i] = new LinkedList<>();
        }
    }

    /** 雜湊函式：除留餘數法 */
    private int hash(int key) {
        return Math.abs(key % TABLE_SIZE);
    }

    /** 插入 */
    public void insert(int key) {
        int index = hash(key);
        if (!buckets[index].contains(key)) {
            buckets[index].add(key);
        }
    }

    /** 查詢 */
    public boolean search(int key) {
        int index = hash(key);
        return buckets[index].contains(key);
    }

    /** 顯示雜湊表內容 */
    public void display() {
        for (int i = 0; i < TABLE_SIZE; i++) {
            System.out.println("桶 " + i + "：" + buckets[i]);
        }
    }

    public static void main(String[] args) {
        SimpleHashTable table = new SimpleHashTable();
        int[] data = {13, 29, 27, 28, 26, 30, 38};

        for (int d : data) {
            table.insert(d);
        }

        table.display();
        System.out.println("查詢 29：" + table.search(29));
        System.out.println("查詢 99：" + table.search(99));
    }
}
```

**輸出**：

```
桶 0：[28]
桶 1：[29]
桶 2：[30]
桶 3：[38]
桶 4：[]
桶 5：[26]
桶 6：[13, 27]
查詢 29：true
查詢 99：false
```

可以看到 13 和 27 都映射到桶 6（`13 % 7 = 6`，`27 % 7 = 6`），透過鏈結法共存於同一個桶中。

## 7、Java 標準庫的查詢工具

Java 標準庫已經內建了多種高效的查詢工具，實務開發中應優先使用：

### Arrays.binarySearch()

對**已排序**的陣列進行二分查詢，回傳元素索引或插入點的負值。

```java
import java.util.Arrays;

int[] arr = {10, 20, 30, 40, 50};
int index = Arrays.binarySearch(arr, 30);   // 回傳 2
int missing = Arrays.binarySearch(arr, 25); // 回傳 -3（插入點為 2，結果為 -(2)-1）
```

### Collections.binarySearch()

對**已排序**的 `List` 進行二分查詢，用法與 `Arrays.binarySearch()` 類似。

```java
import java.util.Collections;
import java.util.List;

List<Integer> list = List.of(10, 20, 30, 40, 50);
int index = Collections.binarySearch(list, 40); // 回傳 3
```

### HashMap — 雜湊查詢的首選

基於雜湊表實作，平均 O(1) 的 `get` / `put` 操作。適合需要快速 key-value 查詢的場景。

```java
import java.util.HashMap;
import java.util.Map;

Map<String, Integer> map = new HashMap<>(16);
map.put("apple", 3);
map.put("banana", 5);
System.out.println(map.get("apple"));       // 3
System.out.println(map.containsKey("cherry")); // false
```

### TreeMap — 有序的 key-value 查詢

基於紅黑樹實作，所有操作 O(log n)，key 自動排序。適合需要**範圍查詢**或**有序遍歷**的場景。

```java
import java.util.TreeMap;

TreeMap<Integer, String> treeMap = new TreeMap<>();
treeMap.put(50, "A");
treeMap.put(30, "B");
treeMap.put(70, "C");

System.out.println(treeMap.firstKey());     // 30（最小 key）
System.out.println(treeMap.floorKey(45));   // 30（<= 45 的最大 key）
System.out.println(treeMap.subMap(30, 70)); // {30=B, 50=A}
```

### 使用時機總結

| 場景 | 推薦工具 |
|------|---------|
| 已排序陣列的單次查詢 | `Arrays.binarySearch()` |
| 已排序 List 的單次查詢 | `Collections.binarySearch()` |
| 大量 key-value 快速查詢 | `HashMap` |
| 需要 key 排序或範圍查詢 | `TreeMap` |
| 需要去重且快速判斷存在 | `HashSet` |
| 需要去重且保持排序 | `TreeSet` |

## 8、小結

五種經典查詢演算法各有適用場景：

- **順序查詢**是基礎，沒有前提條件，但效率最低
- **二分查詢**在已排序資料上效率極高，是面試與實務中最常考的演算法之一
- **索引查詢**適合大量分塊資料，是資料庫索引概念的簡化版
- **BST 查詢**在平衡的情況下等同二分查詢，但需注意退化問題
- **雜湊查詢**平均 O(1)，是實務中最常用的查詢策略

在實際 Java 開發中，善用標準庫的 `Arrays.binarySearch()`、`HashMap`、`TreeMap` 等工具，通常比自行實作更可靠也更高效。理解底層原理的目的，是在需要自訂資料結構或排查效能問題時，能做出正確的判斷。
