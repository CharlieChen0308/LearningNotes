# 01 排序演算法總覽

> **版本**：Java 17+ — 所有程式碼基於 Java 17 編寫

排序是電腦科學中最基礎也最重要的操作之一。無論是資料庫查詢、搜尋引擎索引，還是日常開發中的列表呈現，排序演算法無處不在。本文將系統性地介紹七種經典排序演算法，從時間複雜度、空間複雜度、穩定性等面向進行比較，並附上完整可編譯的 Java 實作。

---

## 1、排序演算法比較表

| 演算法 | 最佳 | 平均 | 最差 | 空間 | 穩定性 |
|--------|------|------|------|------|--------|
| 氣泡排序 | O(n) | O(n²) | O(n²) | O(1) | 穩定 |
| 選擇排序 | O(n²) | O(n²) | O(n²) | O(1) | 不穩定 |
| 插入排序 | O(n) | O(n²) | O(n²) | O(1) | 穩定 |
| 希爾排序 | O(n log n) | O(n^1.3) | O(n²) | O(1) | 不穩定 |
| 歸併排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | 穩定 |
| 快速排序 | O(n log n) | O(n log n) | O(n²) | O(log n) | 不穩定 |
| 堆排序 | O(n log n) | O(n log n) | O(n log n) | O(1) | 不穩定 |

**穩定性**指的是：兩個相等的元素在排序後是否維持原本的相對順序。穩定排序在多欄位排序場景中特別重要——例如先按姓名排序、再按年齡排序時，穩定排序能保證同齡的人依然保持姓名順序。

## 2、O(n²) 排序

O(n²) 排序演算法實作簡單、容易理解，適合小規模資料或教學場景。在資料量超過數千筆時，效能會明顯下降。

### 2.1 氣泡排序（Bubble Sort）

氣泡排序的核心思想是**重複比較相鄰元素**，如果順序錯誤就交換。每一輪遍歷會把當前最大的元素「浮」到陣列尾端，就像氣泡上升一樣。

透過加入 `swapped` 旗標，可以在某一輪完全沒有交換時提前終止，使最佳情況（已排序陣列）的時間複雜度降為 O(n)。

```java
public static void bubbleSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        boolean swapped = false;
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = true;
            }
        }
        // 若本輪沒有任何交換，表示陣列已排序完成
        if (!swapped) break;
    }
}
```

**特性**：穩定排序、原地排序（in-place）。實務上幾乎不會使用，但因概念直觀，常作為教學入門。

### 2.2 選擇排序（Selection Sort）

選擇排序的策略是：每一輪從未排序區間中**找出最小元素**，與未排序區間的第一個元素交換位置。

```java
public static void selectionSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        int minIndex = i;
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIndex]) {
                minIndex = j;
            }
        }
        if (minIndex != i) {
            int temp = arr[i];
            arr[i] = arr[minIndex];
            arr[minIndex] = temp;
        }
    }
}
```

**特性**：不穩定排序（交換可能破壞相等元素的相對順序）、原地排序。無論資料是否已排序，比較次數始終為 n(n-1)/2，因此最佳、平均、最差時間複雜度都是 O(n²)。唯一的優點是**交換次數最少**（最多 n-1 次），在寫入成本高的場景有微小優勢。

### 2.3 插入排序（Insertion Sort）

插入排序將陣列分為「已排序」和「未排序」兩部分。每次從未排序區間取出一個元素，在已排序區間中**找到正確位置插入**。這就像整理撲克牌——每抽一張牌，就插入手中已排好的牌組裡。

```java
public static void insertionSort(int[] arr) {
    int n = arr.length;
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;
        // 將比 key 大的元素逐一往後移
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}
```

**特性**：穩定排序、原地排序。對**近乎已排序的資料特別高效**，時間複雜度趨近 O(n)。在小規模資料（n < 50）上，插入排序往往比 O(n log n) 演算法更快，因為常數因子極低且對快取友善。Java 的 `Arrays.sort()` 內部在子陣列足夠小時，就會切換成插入排序。

## 3、O(n log n) 排序

O(n log n) 是基於比較的排序演算法理論下限。這類演算法能有效處理大規模資料，是實務開發中的主力。

### 3.1 歸併排序（Merge Sort）

歸併排序採用**分治法（Divide and Conquer）**：

1. **分割（Divide）**：將陣列從中點一分為二
2. **遞迴排序（Conquer）**：分別對左半和右半遞迴排序
3. **合併（Merge）**：將兩個已排序的子陣列合併成一個有序陣列

合併過程是歸併排序的核心——用兩個指標分別指向左右子陣列的開頭，每次取較小的元素放入結果陣列，直到所有元素都放完。

```java
public static void mergeSort(int[] arr, int left, int right) {
    if (left >= right) return;

    int mid = left + (right - left) / 2;
    mergeSort(arr, left, mid);
    mergeSort(arr, mid + 1, right);
    merge(arr, left, mid, right);
}

private static void merge(int[] arr, int left, int mid, int right) {
    // 建立左右子陣列的副本
    int[] leftArr = new int[mid - left + 1];
    int[] rightArr = new int[right - mid];

    System.arraycopy(arr, left, leftArr, 0, leftArr.length);
    System.arraycopy(arr, mid + 1, rightArr, 0, rightArr.length);

    int i = 0;  // 左子陣列指標
    int j = 0;  // 右子陣列指標
    int k = left;  // 合併後的寫入位置

    while (i < leftArr.length && j < rightArr.length) {
        if (leftArr[i] <= rightArr[j]) {
            arr[k++] = leftArr[i++];
        } else {
            arr[k++] = rightArr[j++];
        }
    }

    // 將剩餘元素複製回去
    while (i < leftArr.length) {
        arr[k++] = leftArr[i++];
    }
    while (j < rightArr.length) {
        arr[k++] = rightArr[j++];
    }
}
```

**特性**：穩定排序、需要 O(n) 額外空間。時間複雜度在所有情況下都是 O(n log n)，不受輸入資料分佈影響。Java 的 `Arrays.sort()` 對物件陣列使用的 TimSort，其核心便是歸併排序的變體。

### 3.2 快速排序（Quick Sort）

快速排序同樣基於分治法，但策略不同：

1. **選擇基準值（Pivot）**：從陣列中選一個元素作為基準
2. **分區（Partition）**：將小於基準的元素放左邊、大於基準的放右邊
3. **遞迴排序**：對左右兩個分區分別遞迴

以下使用 **Lomuto 分區法**，以陣列最後一個元素為基準：

```java
public static void quickSort(int[] arr, int low, int high) {
    if (low >= high) return;

    int pivotIndex = partition(arr, low, high);
    quickSort(arr, low, pivotIndex - 1);
    quickSort(arr, pivotIndex + 1, high);
}

private static int partition(int[] arr, int low, int high) {
    int pivot = arr[high];
    int i = low - 1;  // i 指向「小於 pivot 區間」的最後一個位置

    for (int j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            i++;
            int temp = arr[i];
            arr[i] = arr[j];
            arr[j] = temp;
        }
    }

    // 將 pivot 放到正確位置
    int temp = arr[i + 1];
    arr[i + 1] = arr[high];
    arr[high] = temp;

    return i + 1;
}
```

**特性**：不穩定排序、原地排序（遞迴堆疊需 O(log n) 空間）。平均時間複雜度 O(n log n)，但在最壞情況下（例如已排序陣列且總是選最後一個元素為基準）會退化成 O(n²)。實務上可透過**隨機選取基準**或**三數取中法（Median of Three）**來避免最壞情況。

快速排序在平均情況下因常數因子小、快取命中率高，通常比歸併排序更快，因此 Java 的 `Arrays.sort()` 對基本型別使用的是**雙基準快速排序（Dual-Pivot Quicksort）**。

### 3.3 堆排序（Heap Sort）

堆排序利用**最大堆積（Max-Heap）**這種資料結構來排序：

1. **建堆（Build Max-Heap）**：將陣列轉化為最大堆積，根節點為最大值
2. **排序**：反覆將堆頂（最大值）與末尾交換，縮小堆的範圍，再調整堆

堆在陣列中的表示方式：對於索引 `i` 的節點，左子節點在 `2i + 1`，右子節點在 `2i + 2`，父節點在 `(i - 1) / 2`。

```java
public static void heapSort(int[] arr) {
    int n = arr.length;

    // 建立最大堆積：從最後一個非葉節點開始，往上逐一調整
    for (int i = n / 2 - 1; i >= 0; i--) {
        heapify(arr, n, i);
    }

    // 逐一將堆頂（最大值）移到陣列尾端
    for (int i = n - 1; i > 0; i--) {
        int temp = arr[0];
        arr[0] = arr[i];
        arr[i] = temp;
        heapify(arr, i, 0);
    }
}

private static void heapify(int[] arr, int heapSize, int rootIndex) {
    int largest = rootIndex;
    int left = 2 * rootIndex + 1;
    int right = 2 * rootIndex + 2;

    if (left < heapSize && arr[left] > arr[largest]) {
        largest = left;
    }
    if (right < heapSize && arr[right] > arr[largest]) {
        largest = right;
    }

    if (largest != rootIndex) {
        int temp = arr[rootIndex];
        arr[rootIndex] = arr[largest];
        arr[largest] = temp;
        heapify(arr, heapSize, largest);
    }
}
```

**特性**：不穩定排序、原地排序（空間 O(1)）。時間複雜度在所有情況下都是 O(n log n)。堆排序的優勢在於**不需要額外空間且沒有最壞情況退化**，但因為存取模式對快取不友善（跳躍式存取），實際速度通常慢於快速排序。常用於需要**嚴格時間上界保證**的場景。

## 4、O(n^1.3) 希爾排序（Shell Sort）

希爾排序是插入排序的改良版本，由 Donald Shell 於 1959 年提出。核心想法是：插入排序在資料近乎已排序時效率很高，那我們就**先讓資料大致有序**，再做最後的插入排序。

做法是引入**間隔序列（Gap Sequence）**：一開始用較大的間隔對相隔較遠的元素進行插入排序，逐步縮小間隔，最終間隔為 1 時等同於標準插入排序。由於前面的大間隔排序已經讓資料大致有序，最後的插入排序只需少量移動。

```java
public static void shellSort(int[] arr) {
    int n = arr.length;

    // 使用 Knuth 間隔序列：1, 4, 13, 40, 121, ...
    int gap = 1;
    while (gap < n / 3) {
        gap = gap * 3 + 1;
    }

    while (gap >= 1) {
        // 對每個間隔執行插入排序
        for (int i = gap; i < n; i++) {
            int key = arr[i];
            int j = i - gap;
            while (j >= 0 && arr[j] > key) {
                arr[j + gap] = arr[j];
                j -= gap;
            }
            arr[j + gap] = key;
        }
        gap /= 3;
    }
}
```

**特性**：不穩定排序、原地排序。時間複雜度取決於間隔序列的選擇，使用 Knuth 序列時平均約為 O(n^1.3)。希爾排序的優勢在於**實作簡單且不需要額外空間**，在中等規模資料上表現不錯，是插入排序到 O(n log n) 演算法之間的實用折衷方案。

## 5、如何選擇排序演算法

在實際 Java 開發中，絕大多數情況應該直接使用標準函式庫：

- **`Arrays.sort(int[])`**：對基本型別陣列使用**雙基準快速排序（Dual-Pivot Quicksort）**，平均效能最佳
- **`Arrays.sort(Object[])`**：對物件陣列使用 **TimSort**（歸併排序 + 插入排序的混合體），保證穩定性
- **`Collections.sort()`**：底層同樣使用 TimSort

那什麼時候需要自行實作排序？以下是選擇指南：

| 場景 | 推薦演算法 | 原因 |
|------|-----------|------|
| 一般開發 | `Arrays.sort()` / `Collections.sort()` | 標準函式庫已高度優化 |
| 小規模資料（n < 50） | 插入排序 | 常數因子最小、快取友善 |
| 需要穩定排序 | 歸併排序 / TimSort | 保證相等元素順序不變 |
| 記憶體受限 | 堆排序 | O(1) 額外空間且無最壞退化 |
| 資料近乎已排序 | 插入排序 | 趨近 O(n) 的表現 |
| 需要嚴格時間上界 | 堆排序 | 最壞情況仍為 O(n log n) |
| 嵌入式/資源受限環境 | 希爾排序 | 不需額外空間、實作簡單、效能合理 |

**重要觀念**：演算法的時間複雜度描述的是增長趨勢，不是實際執行時間。O(n log n) 不一定比 O(n²) 快——當 n 很小時，常數因子和快取命中率更重要。這也是為什麼工業級排序實作（如 TimSort）會在子問題足夠小時切換到插入排序。

## 6、小結

| 分類 | 演算法 | 關鍵特徵 |
|------|--------|---------|
| O(n²) | 氣泡排序 | 教學用途，可提前終止 |
| O(n²) | 選擇排序 | 交換次數最少 |
| O(n²) | 插入排序 | 近乎已排序時極快，小規模首選 |
| O(n^1.3) | 希爾排序 | 插入排序的改良，無需額外空間 |
| O(n log n) | 歸併排序 | 穩定、可預測，需 O(n) 空間 |
| O(n log n) | 快速排序 | 平均最快，Java 基本型別排序首選 |
| O(n log n) | 堆排序 | 原地、無退化，適合嚴格時間限制 |

掌握這些排序演算法不僅是面試基礎，更能幫助你理解「分治」、「堆積」、「漸進分析」等核心觀念，為學習更進階的演算法打下基礎。

**完整測試範例**：

```java
import java.util.Arrays;

public class SortingDemo {

    // === 氣泡排序 ===
    public static void bubbleSort(int[] arr) {
        int n = arr.length;
        for (int i = 0; i < n - 1; i++) {
            boolean swapped = false;
            for (int j = 0; j < n - 1 - i; j++) {
                if (arr[j] > arr[j + 1]) {
                    int temp = arr[j];
                    arr[j] = arr[j + 1];
                    arr[j + 1] = temp;
                    swapped = true;
                }
            }
            if (!swapped) break;
        }
    }

    // === 選擇排序 ===
    public static void selectionSort(int[] arr) {
        int n = arr.length;
        for (int i = 0; i < n - 1; i++) {
            int minIndex = i;
            for (int j = i + 1; j < n; j++) {
                if (arr[j] < arr[minIndex]) {
                    minIndex = j;
                }
            }
            if (minIndex != i) {
                int temp = arr[i];
                arr[i] = arr[minIndex];
                arr[minIndex] = temp;
            }
        }
    }

    // === 插入排序 ===
    public static void insertionSort(int[] arr) {
        int n = arr.length;
        for (int i = 1; i < n; i++) {
            int key = arr[i];
            int j = i - 1;
            while (j >= 0 && arr[j] > key) {
                arr[j + 1] = arr[j];
                j--;
            }
            arr[j + 1] = key;
        }
    }

    // === 希爾排序 ===
    public static void shellSort(int[] arr) {
        int n = arr.length;
        int gap = 1;
        while (gap < n / 3) {
            gap = gap * 3 + 1;
        }
        while (gap >= 1) {
            for (int i = gap; i < n; i++) {
                int key = arr[i];
                int j = i - gap;
                while (j >= 0 && arr[j] > key) {
                    arr[j + gap] = arr[j];
                    j -= gap;
                }
                arr[j + gap] = key;
            }
            gap /= 3;
        }
    }

    // === 歸併排序 ===
    public static void mergeSort(int[] arr, int left, int right) {
        if (left >= right) return;
        int mid = left + (right - left) / 2;
        mergeSort(arr, left, mid);
        mergeSort(arr, mid + 1, right);
        merge(arr, left, mid, right);
    }

    private static void merge(int[] arr, int left, int mid, int right) {
        int[] leftArr = new int[mid - left + 1];
        int[] rightArr = new int[right - mid];
        System.arraycopy(arr, left, leftArr, 0, leftArr.length);
        System.arraycopy(arr, mid + 1, rightArr, 0, rightArr.length);

        int i = 0, j = 0, k = left;
        while (i < leftArr.length && j < rightArr.length) {
            if (leftArr[i] <= rightArr[j]) {
                arr[k++] = leftArr[i++];
            } else {
                arr[k++] = rightArr[j++];
            }
        }
        while (i < leftArr.length) arr[k++] = leftArr[i++];
        while (j < rightArr.length) arr[k++] = rightArr[j++];
    }

    // === 快速排序 ===
    public static void quickSort(int[] arr, int low, int high) {
        if (low >= high) return;
        int pivotIndex = partition(arr, low, high);
        quickSort(arr, low, pivotIndex - 1);
        quickSort(arr, pivotIndex + 1, high);
    }

    private static int partition(int[] arr, int low, int high) {
        int pivot = arr[high];
        int i = low - 1;
        for (int j = low; j < high; j++) {
            if (arr[j] <= pivot) {
                i++;
                int temp = arr[i];
                arr[i] = arr[j];
                arr[j] = temp;
            }
        }
        int temp = arr[i + 1];
        arr[i + 1] = arr[high];
        arr[high] = temp;
        return i + 1;
    }

    // === 堆排序 ===
    public static void heapSort(int[] arr) {
        int n = arr.length;
        for (int i = n / 2 - 1; i >= 0; i--) {
            heapify(arr, n, i);
        }
        for (int i = n - 1; i > 0; i--) {
            int temp = arr[0];
            arr[0] = arr[i];
            arr[i] = temp;
            heapify(arr, i, 0);
        }
    }

    private static void heapify(int[] arr, int heapSize, int rootIndex) {
        int largest = rootIndex;
        int left = 2 * rootIndex + 1;
        int right = 2 * rootIndex + 2;
        if (left < heapSize && arr[left] > arr[largest]) largest = left;
        if (right < heapSize && arr[right] > arr[largest]) largest = right;
        if (largest != rootIndex) {
            int temp = arr[rootIndex];
            arr[rootIndex] = arr[largest];
            arr[largest] = temp;
            heapify(arr, heapSize, largest);
        }
    }

    // === 測試入口 ===
    public static void main(String[] args) {
        int[] original = {38, 27, 43, 3, 9, 82, 10};

        // 測試每種排序
        String[] names = {
            "氣泡排序", "選擇排序", "插入排序", "希爾排序",
            "歸併排序", "快速排序", "堆排序"
        };

        for (String name : names) {
            int[] arr = Arrays.copyOf(original, original.length);
            switch (name) {
                case "氣泡排序" -> bubbleSort(arr);
                case "選擇排序" -> selectionSort(arr);
                case "插入排序" -> insertionSort(arr);
                case "希爾排序" -> shellSort(arr);
                case "歸併排序" -> mergeSort(arr, 0, arr.length - 1);
                case "快速排序" -> quickSort(arr, 0, arr.length - 1);
                case "堆排序"  -> heapSort(arr);
            }
            System.out.printf("%-8s: %s%n", name, Arrays.toString(arr));
        }
    }
}
```
