# 03 基礎資料結構

> **版本**：Java 17+ — 所有程式碼基於 Java 17 編寫

本篇整合陣列、鏈結串列、棧、佇列、雙端佇列五種基礎線性資料結構，從概念、複雜度分析到 Java 實作，一篇掌握核心知識。

## 1、複雜度速查表

| 資料結構 | 存取 | 搜尋 | 插入 | 刪除 | Java 對應 |
|---------|------|------|------|------|----------|
| 陣列（Array） | O(1) | O(n) | O(n) | O(n) | `int[]`、`ArrayList` |
| 鏈結串列（Linked List） | O(n) | O(n) | O(1)* | O(1)* | `LinkedList` |
| 棧（Stack） | O(n) | O(n) | O(1) | O(1) | `ArrayDeque` |
| 佇列（Queue） | O(n) | O(n) | O(1) | O(1) | `ArrayDeque`、`LinkedList` |
| 雙端佇列（Deque） | O(n) | O(n) | O(1) | O(1) | `ArrayDeque` |

> *鏈結串列的插入／刪除為 O(1) 是指「已定位到節點」的情況；若需先搜尋再操作，整體為 O(n)。

## 2、陣列與順序表（Array / ArrayList）

### 2.1 固定大小陣列 vs 動態陣列

**固定大小陣列**在宣告時就決定了長度，記憶體連續配置，透過索引直接存取元素，時間複雜度 O(1)。缺點是長度不可變，插入或刪除中間元素需要搬移後續所有元素，時間複雜度 O(n)。

**動態陣列**（如 `ArrayList`）在底層仍然使用固定大小陣列，但在容量不足時會自動擴容（通常為 1.5 倍），對外提供可變長度的介面。

### 2.2 時間複雜度分析

| 操作 | 時間複雜度 | 說明 |
|------|-----------|------|
| 隨機存取 `get(i)` | O(1) | 直接根據索引計算記憶體位址 |
| 尾端新增 `add(e)` | 均攤 O(1) | 偶爾觸發擴容為 O(n)，但均攤後為 O(1) |
| 指定位置插入 `add(i, e)` | O(n) | 需搬移 `n - i` 個元素 |
| 指定位置刪除 `remove(i)` | O(n) | 需搬移 `n - i - 1` 個元素 |
| 搜尋 `contains(e)` | O(n) | 線性掃描 |

### 2.3 Java 實作：簡易動態陣列

以下實作展示核心的擴容邏輯：

```java
public class SimpleArrayList<E> {

    private static final int DEFAULT_CAPACITY = 10;

    private Object[] elements;
    private int size;

    public SimpleArrayList() {
        elements = new Object[DEFAULT_CAPACITY];
    }

    public int size() {
        return size;
    }

    @SuppressWarnings("unchecked")
    public E get(int index) {
        checkIndex(index);
        return (E) elements[index];
    }

    public void add(E element) {
        ensureCapacity();
        elements[size++] = element;
    }

    public void add(int index, E element) {
        if (index < 0 || index > size) {
            throw new IndexOutOfBoundsException("Index: " + index);
        }
        ensureCapacity();
        System.arraycopy(elements, index, elements, index + 1, size - index);
        elements[index] = element;
        size++;
    }

    @SuppressWarnings("unchecked")
    public E remove(int index) {
        checkIndex(index);
        E removed = (E) elements[index];
        int moveCount = size - index - 1;
        if (moveCount > 0) {
            System.arraycopy(elements, index + 1, elements, index, moveCount);
        }
        elements[--size] = null; // 幫助 GC
        return removed;
    }

    private void ensureCapacity() {
        if (size == elements.length) {
            int newCapacity = elements.length + (elements.length >> 1); // 1.5 倍
            elements = java.util.Arrays.copyOf(elements, newCapacity);
        }
    }

    private void checkIndex(int index) {
        if (index < 0 || index >= size) {
            throw new IndexOutOfBoundsException("Index: " + index);
        }
    }
}
```

> **延伸**：Java 標準庫的 `java.util.ArrayList` 採用相同的擴容策略（1.5 倍），並額外處理了序列化、`Iterator`、`SubList` 等功能。

## 3、鏈結串列（Linked List）

### 3.1 單向鏈結串列

每個節點包含資料域與指向下一節點的參考。不需要連續記憶體，插入和刪除只需修改指標，不需搬移元素。

```java
public class SinglyLinkedList<E> {

    private static class Node<E> {
        E data;
        Node<E> next;

        Node(E data) {
            this.data = data;
        }
    }

    private Node<E> head;
    private int size;

    public int size() {
        return size;
    }

    public boolean isEmpty() {
        return size == 0;
    }

    /** 在頭部插入，O(1) */
    public void addFirst(E element) {
        var node = new Node<>(element);
        node.next = head;
        head = node;
        size++;
    }

    /** 在尾部插入，O(n) */
    public void addLast(E element) {
        var node = new Node<>(element);
        if (head == null) {
            head = node;
        } else {
            var current = head;
            while (current.next != null) {
                current = current.next;
            }
            current.next = node;
        }
        size++;
    }

    /** 刪除第一個匹配的元素，O(n) */
    public boolean remove(E element) {
        if (head == null) return false;

        if (java.util.Objects.equals(head.data, element)) {
            head = head.next;
            size--;
            return true;
        }

        var current = head;
        while (current.next != null) {
            if (java.util.Objects.equals(current.next.data, element)) {
                current.next = current.next.next;
                size--;
                return true;
            }
            current = current.next;
        }
        return false;
    }

    /** 搜尋，O(n) */
    public boolean contains(E element) {
        var current = head;
        while (current != null) {
            if (java.util.Objects.equals(current.data, element)) {
                return true;
            }
            current = current.next;
        }
        return false;
    }

    /** 列印所有元素 */
    public void printAll() {
        var current = head;
        while (current != null) {
            System.out.print(current.data + " -> ");
            current = current.next;
        }
        System.out.println("null");
    }
}
```

### 3.2 雙向鏈結串列

雙向鏈結串列（Doubly Linked List）的每個節點同時持有 `prev` 和 `next` 指標，可以雙向遍歷。Java 的 `java.util.LinkedList` 就是雙向鏈結串列的實作，同時實現了 `List` 和 `Deque` 介面。

### 3.3 常見面試題

**反轉鏈結串列**：使用三指標法，逐一反轉每個節點的 `next` 指向。

```java
public class LinkedListProblems {

    private record ListNode(int val, ListNode next) {
        ListNode(int val) { this(val, null); }
    }

    /** 反轉單向鏈結串列，O(n) 時間、O(1) 空間 */
    static ListNode reverse(ListNode head) {
        ListNode prev = null;
        ListNode current = head;
        while (current != null) {
            var next = current.next;
            current = new ListNode(current.val, prev); // record 不可變，重建節點
            prev = current;
            current = next;
        }
        return prev;
    }
}
```

> 如果使用可變的 `Node` 類別，可以直接修改 `next` 欄位而不需重建節點。

**偵測環（Floyd 龜兔演算法）**：使用快慢兩個指標，慢指標每次走一步，快指標每次走兩步。若鏈結串列有環，兩者必定相遇；若快指標到達 `null`，則無環。

```java
public class CycleDetection {

    static class ListNode {
        int val;
        ListNode next;
        ListNode(int val) { this.val = val; }
    }

    /** Floyd 龜兔演算法，O(n) 時間、O(1) 空間 */
    static boolean hasCycle(ListNode head) {
        ListNode slow = head;
        ListNode fast = head;
        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;
            if (slow == fast) return true;
        }
        return false;
    }
}
```

> **延伸**：Java 標準庫的 `java.util.LinkedList` 是雙向鏈結串列，支援 `List`、`Deque`、`Queue` 介面，但隨機存取效能為 O(n)，不適合需要頻繁按索引存取的場景。

## 4、棧（Stack）

### 4.1 LIFO 概念

棧（Stack）是一種**後進先出**（Last In, First Out）的線性表。只允許在同一端（稱為棧頂）進行插入（push）和刪除（pop）。可以類比為一疊盤子：只能從最上方放入或取出。

### 4.2 Java 實作：陣列棧

```java
public class ArrayStack<E> {

    private static final int DEFAULT_CAPACITY = 16;

    private Object[] elements;
    private int top; // 指向下一個可放入的位置

    public ArrayStack() {
        elements = new Object[DEFAULT_CAPACITY];
    }

    public boolean isEmpty() {
        return top == 0;
    }

    public int size() {
        return top;
    }

    public void push(E element) {
        if (top == elements.length) {
            elements = java.util.Arrays.copyOf(elements, elements.length * 2);
        }
        elements[top++] = element;
    }

    @SuppressWarnings("unchecked")
    public E pop() {
        if (isEmpty()) throw new java.util.NoSuchElementException("Stack is empty");
        E element = (E) elements[--top];
        elements[top] = null; // 幫助 GC
        return element;
    }

    @SuppressWarnings("unchecked")
    public E peek() {
        if (isEmpty()) throw new java.util.NoSuchElementException("Stack is empty");
        return (E) elements[top - 1];
    }
}
```

### 4.3 應用：括號匹配

棧最經典的應用之一是檢查括號是否正確配對：

```java
public class BracketMatcher {

    public static boolean isValid(String s) {
        var stack = new java.util.ArrayDeque<Character>();
        for (char c : s.toCharArray()) {
            switch (c) {
                case '(', '[', '{' -> stack.push(c);
                case ')' -> { if (stack.isEmpty() || stack.pop() != '(') return false; }
                case ']' -> { if (stack.isEmpty() || stack.pop() != '[') return false; }
                case '}' -> { if (stack.isEmpty() || stack.pop() != '{') return false; }
                default -> { /* 忽略非括號字元 */ }
            }
        }
        return stack.isEmpty();
    }
}
```

### 4.4 應用：後綴表達式求值

棧也用於計算後綴表達式（逆波蘭表示法）。遇到數字就 push，遇到運算子就 pop 兩個運算元，計算後 push 回去。

### 4.5 Java 中的棧

Java 有一個 `java.util.Stack` 類別，但官方文件建議使用 `ArrayDeque` 替代：

```java
// 不推薦
java.util.Stack<Integer> oldStack = new java.util.Stack<>();

// 推薦：ArrayDeque 作為棧使用
Deque<Integer> stack = new java.util.ArrayDeque<>();
stack.push(1);
stack.push(2);
int top = stack.pop(); // 2
```

原因：`Stack` 繼承自 `Vector`，所有方法都加了 `synchronized`，在單執行緒場景下有不必要的效能開銷。`ArrayDeque` 無鎖、基於陣列，效能更好。

## 5、佇列（Queue）

### 5.1 FIFO 概念

佇列（Queue）是一種**先進先出**（First In, First Out）的線性表。元素從隊尾（rear）進入，從隊頭（front）離開，就像排隊買東西一樣。

### 5.2 Java 實作：循環佇列

普通陣列佇列在出隊時需要搬移所有元素，效率低。**循環佇列**利用取餘運算讓 `front` 和 `rear` 在陣列中循環移動，避免搬移，是佇列實作的重點。

```java
public class CircularQueue<E> {

    private final Object[] elements;
    private int front; // 指向隊頭元素
    private int rear;  // 指向隊尾的下一個空位
    private int size;

    public CircularQueue(int capacity) {
        elements = new Object[capacity];
    }

    public boolean isEmpty() {
        return size == 0;
    }

    public boolean isFull() {
        return size == elements.length;
    }

    public int size() {
        return size;
    }

    /**
     * 入隊：將元素放在 rear 位置，rear 向後移動一格（循環）。
     */
    public boolean enqueue(E element) {
        if (isFull()) return false;
        elements[rear] = element;
        rear = (rear + 1) % elements.length; // 關鍵：取餘實現循環
        size++;
        return true;
    }

    /**
     * 出隊：取出 front 位置的元素，front 向後移動一格（循環）。
     */
    @SuppressWarnings("unchecked")
    public E dequeue() {
        if (isEmpty()) throw new java.util.NoSuchElementException("Queue is empty");
        E element = (E) elements[front];
        elements[front] = null;
        front = (front + 1) % elements.length; // 關鍵：取餘實現循環
        size--;
        return element;
    }

    /**
     * 查看隊頭元素但不移除。
     */
    @SuppressWarnings("unchecked")
    public E peek() {
        if (isEmpty()) throw new java.util.NoSuchElementException("Queue is empty");
        return (E) elements[front];
    }
}
```

循環佇列的核心在於 `front` 和 `rear` 指標的移動邏輯：

- **入隊**：`rear = (rear + 1) % capacity`
- **出隊**：`front = (front + 1) % capacity`
- **判滿**：使用獨立的 `size` 計數器（另一種常見做法是預留一個空位，用 `(rear + 1) % capacity == front` 判滿）

### 5.3 優先佇列

**優先佇列**（Priority Queue）不遵循 FIFO，而是按照元素的優先級出隊。底層通常使用堆（Heap）實現。Java 提供了 `java.util.PriorityQueue`，預設為最小堆：

```java
var pq = new java.util.PriorityQueue<Integer>();
pq.offer(30);
pq.offer(10);
pq.offer(20);
System.out.println(pq.poll()); // 10（最小值先出）
```

### 5.4 Java 的 Queue 介面

`java.util.Queue` 介面定義了佇列的標準操作：

| 方法（拋例外） | 方法（回傳特殊值） | 說明 |
|--------------|------------------|------|
| `add(e)` | `offer(e)` | 入隊 |
| `remove()` | `poll()` | 出隊 |
| `element()` | `peek()` | 查看隊頭 |

常用實作類別：
- **`ArrayDeque`**：基於循環陣列，效能最佳的通用佇列
- **`LinkedList`**：基於雙向鏈結串列，同時實作 `List` 和 `Deque`
- **`PriorityQueue`**：基於二元堆的優先佇列

## 6、雙端佇列（Deque）

### 6.1 概念

雙端佇列（Double-Ended Queue，簡稱 Deque）允許在**兩端**同時進行插入和刪除操作。它同時具備棧和佇列的能力。

### 6.2 Java 的 ArrayDeque

`java.util.ArrayDeque` 是 Java 中最推薦的 Deque 實作，底層為可變大小的循環陣列。官方文件明確指出：

> "This class is likely to be faster than Stack when used as a stack, and faster than LinkedList when used as a queue."

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class DequeDemo {

    public static void main(String[] args) {
        Deque<String> deque = new ArrayDeque<>();

        // 當作佇列使用（FIFO）
        deque.offerLast("A");
        deque.offerLast("B");
        deque.offerLast("C");
        System.out.println(deque.pollFirst()); // A

        // 當作棧使用（LIFO）
        deque.push("X"); // 等同 addFirst
        deque.push("Y");
        System.out.println(deque.pop());       // Y（等同 removeFirst）

        // 雙端操作
        deque.offerFirst("HEAD");
        deque.offerLast("TAIL");
        System.out.println(deque.peekFirst()); // HEAD
        System.out.println(deque.peekLast());  // TAIL
    }
}
```

### 6.3 為什麼 ArrayDeque 優於 Stack 和 LinkedList

| 比較項目 | ArrayDeque | Stack | LinkedList |
|---------|-----------|-------|-----------|
| 底層結構 | 循環陣列 | 陣列（Vector） | 雙向鏈結串列 |
| 執行緒安全 | 否（快） | 是（慢，synchronized） | 否 |
| 記憶體使用 | 緊湊、快取友善 | 緊湊 | 每個節點額外兩個指標 |
| 效能 | 最佳 | 鎖的開銷 | 指標追蹤、快取不友善 |

結論：在單執行緒環境下，**一律使用 `ArrayDeque`** 作為棧和佇列的實作。

## 7、Java Collections Framework 對照

| 資料結構概念 | Java 類別 | 特點 |
|------------|----------|------|
| 動態陣列 | `ArrayList` | 隨機存取 O(1)，尾端增刪均攤 O(1) |
| 雙向鏈結串列 | `LinkedList` | 頭尾增刪 O(1)，隨機存取 O(n) |
| 棧 / 佇列 / 雙端佇列 | `ArrayDeque` | 基於循環陣列，兩端操作 O(1)，推薦首選 |
| 優先佇列 | `PriorityQueue` | 基於二元堆，`poll()` 取最小值 O(log n) |
| 雜湊表 | `HashMap` | 鍵值對存取均攤 O(1)，無序 |
| 有序映射 | `TreeMap` | 基於紅黑樹，鍵值對操作 O(log n)，按鍵排序 |

選擇建議：

- **需要按索引存取** → `ArrayList`
- **頻繁在頭尾增刪** → `ArrayDeque`
- **需要棧或佇列** → `ArrayDeque`
- **需要優先級排序** → `PriorityQueue`
- **鍵值查找** → `HashMap`（無序）或 `TreeMap`（有序）

## 8、小結

五種基礎資料結構各有適用場景：

- **陣列 / ArrayList**：需要快速隨機存取時的首選，但中間位置的插入刪除代價高。
- **鏈結串列**：適合頻繁插入刪除且不需要隨機存取的場景，面試中的高頻考點。
- **棧**：處理具有「後進先出」特性的問題，如括號匹配、函式呼叫堆疊、表達式求值。
- **佇列**：處理「先進先出」的任務排程，如 BFS、訊息佇列、緩衝區。
- **雙端佇列**：同時需要頭尾操作時使用，`ArrayDeque` 是 Java 中最通用的線性結構實作。

理解這些基礎結構的時間複雜度與適用場景，是學習更進階資料結構（樹、圖、堆）和演算法的基石。
