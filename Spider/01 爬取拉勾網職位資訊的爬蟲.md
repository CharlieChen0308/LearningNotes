## 思路

1、首先我們開啟拉勾網，並搜尋“java”，顯示出來的職位資訊就是我們的目標。

2、接下來我們需要確定，怎樣將資訊提取出來。

* 檢視網頁原始碼，這時候發現，網頁原始碼裡面找不到職位相關資訊，這證明拉勾閘道器於職位的資訊是非同步載入的，這也是一種很常用的技術。

* 非同步載入的資訊，我們需要藉助 chrome 瀏覽器的開發者工具進行分析，開啟開發者工具的方法如下：

![][1]

* 點選Nerwork進入網路分析介面，這時候是一片空白，重新整理一下介面就可以看到一系列的網路請求了。

![][2]

* 前面我們說到，拉勾閘道器於職位的資訊是非同步載入的，那麼在這一系列的網路請求中，必定有某個請求傳送給伺服器，響應回來的是職位資訊。

* 正常情況下，我們可以忽略css，圖片等型別的請求，關注點放在XHR這種型別請求上，如圖：

![][3]

一共4個XHR型別的請求，我們逐個開啟對比，分別點選Preview就能看到它們響應的內容。

發現第一個請求就是我們要找的。如圖：

**3、資訊解析部分`parse.py`**

class Parse:
    '''
    解析網頁資訊
    '''

    def __init__(self, htmlCode):
        self.htmlCode = htmlCode
        self.json = demjson.decode(htmlCode)
        pass

    def parseTool(self, content):
        '''
        清除html標籤
        '''
        if type(content) != str: return content
        sublist = ['<p.*?>', '</p.*?>', '<b.*?>', '</b.*?>', '<div.*?>', '</div.*?>',
                   '</br>', '<br />', '<ul>', '</ul>', '<li>', '</li>', '<strong>',
                   '</strong>', '<table.*?>', '<tr.*?>', '</tr>', '<td.*?>', '</td>',
                   '\r', '\n', '&.*?;', '&', '#.*?;', '<em>', '</em>']
        try:
            for substring in [re.compile(string, re.S) for string in sublist]:
                content = re.sub(substring, "", content).strip()
        except:
            raise Exception('Error ' + str(substring.pattern))
        return content
