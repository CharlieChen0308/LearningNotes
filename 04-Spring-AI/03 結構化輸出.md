# 03 結構化輸出（Structured Output）

> **版本**：Spring AI 1.0+ / Spring Boot 3.4.x / Java 17+

## 為什麼需要結構化輸出

AI 模型的回應通常是自由格式的文字，但在實際應用中，我們經常需要將 AI 的回應解析為結構化的 Java 物件，以便後續處理。例如：

- 從產品描述中提取規格資訊
- 將自然語言轉換為 JSON 格式
- 讓 AI 回傳特定欄位的資料

Spring AI 提供了 Structured Output 機制，可以自動將 AI 回應對映為 Java POJO。

## 基本使用：entity() 方法

### 定義資料類別

```java
public record ActorsFilms(
    String actor,
    List<String> movies
) {}
```

### 自動解析為物件

```java
ActorsFilms result = chatClient.prompt()
    .user("請列出乘風破浪的主要演員和他們的代表作品")
    .call()
    .entity(ActorsFilms.class);

System.out.println(result.actor());    // 鄧超
System.out.println(result.movies());   // [美人魚, 乘風破浪, ...]
```

### 解析為 List

```java
List<ActorsFilms> results = chatClient.prompt()
    .user("請列出 3 位台灣知名演員和他們的代表作品")
    .call()
    .entity(new ParameterizedTypeReference<List<ActorsFilms>>() {});
```

## 原生結構化輸出（Native Structured Output）

某些 AI 模型原生支援 JSON Schema 輸出，Spring AI 可以直接利用此功能，提高解析可靠性：

```java
// 透過 ChatOptions 啟用原生結構化輸出（以 OpenAI 為例）
ActorsFilms result = ChatClient.create(chatModel).prompt()
    .options(OpenAiChatOptions.builder()
        .responseFormat(new ResponseFormat(ResponseFormat.Type.JSON_SCHEMA))
        .build())
    .user("Generate the filmography for a random actor.")
    .call()
    .entity(ActorsFilms.class);
```

> **API 版本注意**：早期預覽版使用 `AdvisorParams.ENABLE_NATIVE_STRUCTURED_OUTPUT` 常數啟用原生模式，
> Spring AI 1.0.0 GA 後該常數可能已移除或重新命名。請以官方文件為準，確認目標版本啟用原生結構化輸出的方式。
> 部分模型提供商（如 OpenAI）可透過 `ChatOptions` 的 `responseFormat` 設定 JSON Schema 模式。

使用原生模式時，Spring AI 會將 `BeanOutputConverter` 產生的 JSON Schema 直接傳送給模型的結構化輸出 API，模型保證輸出符合格式的 JSON，不再需要在 prompt 中附加格式說明。

### 模型支援狀況

並非所有模型都支援原生結構化輸出，各模型的支援方式不同：

| 模型 | 原生 JSON Schema | 說明 |
|------|:-:|------|
| OpenAI（GPT-4o、GPT-4o-mini） | ✓ | 透過 `response_format` 的 `json_schema` 類型，可靠度最高 |
| Ollama（Llama 3.2 等） | ✓ | 透過 `format` 參數傳入 JSON Schema Map |
| Anthropic Claude | ✗ | 僅支援 prompt-based 提取，透過 prompt 指示格式 |
| Google Gemini | 部分 | 部分版本支援，建議查閱最新文件 |

> **建議**：如果目標模型支援原生 JSON Schema，優先透過 `ChatOptions` 的 `responseFormat` 啟用，可顯著提升解析成功率。不支援的模型會自動降級為 prompt-based 方式。

## 使用 OutputConverter

### BeanOutputConverter

用於將 AI 回應轉換為 Java Bean：

```java
public record BookInfo(
    String title,
    String author,
    int year,
    String summary
) {}
```

```java
BeanOutputConverter<BookInfo> converter = new BeanOutputConverter<>(BookInfo.class);

String format = converter.getFormat();  // 生成格式說明
Prompt prompt = new Prompt(
    "請推薦一本 Java 程式設計書籍，用以下格式回答：" + format);

String response = chatClient.prompt(prompt)
    .call()
    .content();

BookInfo book = converter.convert(response);
```

### ListOutputConverter

將回應轉換為字串列表：

```java
ListOutputConverter converter = new ListOutputConverter(
    new DefaultConversionService());

String format = converter.getFormat();
String response = chatClient.prompt()
    .user("請列出 Java 17 的 5 個重要新特性。" + format)
    .call()
    .content();

List<String> features = converter.convert(response);
```

### MapOutputConverter

將回應轉換為 Map：

```java
MapOutputConverter converter = new MapOutputConverter();

String format = converter.getFormat();
String response = chatClient.prompt()
    .user("請比較 Java 和 Kotlin 的主要差異，用鍵值對方式呈現。" + format)
    .call()
    .content();

Map<String, Object> comparison = converter.convert(response);
```

## 方式比較與選型建議

### entity() vs OutputConverter

| 面向 | `entity()` 方法 | `OutputConverter` |
|------|----------------|-------------------|
| 複雜度 | 低，一行呼叫 | 中，需手動組合 prompt 與轉換 |
| 適用場景 | 標準 POJO / List 對映 | 自訂格式、Map、需要控制 prompt 內容 |
| 原生模式 | 搭配 `ChatOptions.responseFormat` 即可啟用 | 需自行設定模型的 `responseFormat` |
| **建議** | **優先使用**，涵蓋大多數場景 | 當 `entity()` 無法滿足需求時使用 |

### 原生 JSON Schema vs Prompt-based

| 面向 | 原生 JSON Schema | Prompt-based（預設） |
|------|-----------------|---------------------|
| 可靠度 | 高，模型保證輸出符合 Schema | 中，模型可能偏離格式 |
| 模型依賴 | 僅支援特定模型（OpenAI、Ollama 等） | 通用，所有模型皆可使用 |
| Prompt 乾淨度 | 不汙染 prompt | 會在 prompt 中附加格式說明 |
| **建議** | 目標模型支援時優先啟用 | 作為通用 fallback 方案 |

## 複雜結構範例

### 巢狀結構

```java
public record CourseOutline(
    String courseName,
    String description,
    List<Chapter> chapters
) {
    public record Chapter(
        int order,
        String title,
        List<String> topics,
        int estimatedMinutes
    ) {}
}
```

```java
CourseOutline outline = chatClient.prompt()
    .user("請設計一個 Spring Boot 入門課程大綱，包含 5 個章節")
    .call()
    .entity(CourseOutline.class);

outline.chapters().forEach(chapter -> {
    System.out.println(chapter.order() + ". " + chapter.title());
    chapter.topics().forEach(topic ->
        System.out.println("   - " + topic));
});
```

### 使用 @JsonPropertyDescription 增加欄位描述

```java
public record ProductAnalysis(
    @JsonPropertyDescription("產品名稱")
    String name,

    @JsonPropertyDescription("產品類別，如：電子、食品、服飾")
    String category,

    @JsonPropertyDescription("正面評價的關鍵字列表")
    List<String> positiveKeywords,

    @JsonPropertyDescription("負面評價的關鍵字列表")
    List<String> negativeKeywords,

    @JsonPropertyDescription("總體評分，1-10 分")
    int overallScore
) {}
```

## 生產環境注意事項

### 解析失敗處理與重試

AI 模型有時可能回傳不符合預期格式的內容，生產環境中必須加上錯誤處理與重試機制：

```java
private static final int MAX_RETRIES = 3;

public <T> T callWithRetry(String userMessage, Class<T> type) {
    for (int attempt = 1; attempt <= MAX_RETRIES; attempt++) {
        try {
            return chatClient.prompt()
                .user(userMessage)
                .call()
                .entity(type);
        } catch (Exception e) {
            log.warn("結構化輸出解析失敗（第 {} 次）：{}", attempt, e.getMessage());
            if (attempt == MAX_RETRIES) {
                throw new StructuredOutputException(
                    "重試 " + MAX_RETRIES + " 次後仍無法解析", e);
            }
        }
    }
    throw new IllegalStateException("不應到達此處");
}
```

### JSON Schema 複雜度限制

- **避免過深的巢狀結構**：建議不超過 3～4 層巢狀，層數越深模型越容易產生格式錯誤
- **控制欄位數量**：單一物件建議不超過 15～20 個欄位，過多欄位會降低解析成功率
- **善用 `@JsonPropertyDescription`**：為每個欄位加上清晰的描述，幫助模型理解欄位語意
- **避免使用泛型或過於抽象的型別**：如 `Object`、`Map<String, Object>` 等，會導致 Schema 不明確

### 輸出驗證

即使解析成功，仍建議對結構化輸出進行業務層驗證：

```java
ActorsFilms result = chatClient.prompt()
    .user("請列出周星馳的代表作品")
    .call()
    .entity(ActorsFilms.class);

// 驗證必要欄位
if (result.actor() == null || result.actor().isBlank()) {
    throw new ValidationException("actor 欄位不得為空");
}
if (result.movies() == null || result.movies().isEmpty()) {
    throw new ValidationException("movies 列表不得為空");
}
```

> **提醒**：結構化輸出保證的是「格式正確」，不保證「內容正確」。AI 仍可能產生幻覺（hallucination），關鍵業務資料應搭配其他驗證機制。

## 小結

結構化輸出是 Spring AI 的強大功能，讓 AI 回應可以無縫對映到 Java 物件。搭配 `entity()` 方法和原生 JSON Schema 支援，開發者能夠可靠地從 AI 模型獲取結構化資料，輕鬆整合到業務邏輯中。
