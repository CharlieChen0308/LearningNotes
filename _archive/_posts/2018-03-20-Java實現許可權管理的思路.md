## 許可權管理分析

每個網站都涉及到訪問許可權的控制。每個站點資源都需要被管理起來，使用者只有具有訪問某個資源的特定許可權，才能夠訪問，否則就被拒絕訪問。網站的訪問許可權控制，一種方法從 URL 入手，站點的每個資源都用唯一的 URL 描述，我們為想要管理起來的 URL 增加上許可權屬性，當使用者訪問資源時我們要先檢查使用者是否具有許可權。採用攔截器技術實現許可權攔截，進行判斷，有許可權即可訪問，沒有就阻止。另一種方法是採用入口有無的形式來做，根據後臺資料判斷有許可權則在介面上有許可權入口，沒有許可權則沒有入口。

## 許可權資料庫設計

資料庫採用如下設計，總共由5張表組成，使用者表、角色表、資源表、使用者角色表和角色資源表。一個使用者可以擁有多個角色，一個角色擁有多個資源。

![][1]

```sql
CREATE TABLE `user` (
  `id` int(11) NOT NULL AUTO_INCREMENT COMMENT 'id',
  `username` varchar(32) DEFAULT NULL COMMENT '使用者名稱',
  `password` varchar(32) DEFAULT NULL COMMENT '密碼',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8 COMMENT='使用者表';

CREATE TABLE `role` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `name` varchar(32) DEFAULT NULL COMMENT '角色名',
  `desc` varchar(32) DEFAULT NULL COMMENT '角色描述',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8 COMMENT='角色表';

CREATE TABLE `resource` (
  `id` int(11) NOT NULL AUTO_INCREMENT COMMENT 'id',
  `title` varchar(32) DEFAULT NULL COMMENT '資源標題',
  `uri` varchar(32) DEFAULT NULL COMMENT '資源uri  ',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8 COMMENT='資源表';

CREATE TABLE `user_role` (
  `id` int(11) NOT NULL AUTO_INCREMENT COMMENT 'id',
  `user_id` int(11) NOT NULL COMMENT '使用者id',
  `role_id` int(11) NOT NULL COMMENT '角色id',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8 COMMENT='使用者角色表';

CREATE TABLE `role_resource` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `role_id` int(11) DEFAULT NULL COMMENT '角色id',
  `resource_id` int(11) DEFAULT NULL COMMENT '資源id',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8 COMMENT='角色資源表';


```

## 攔截URL實現許可權管理

透過 URL 進行請求，由攔截器進行攔截，根據使用者的資訊，去資料庫中查詢此使用者是否具有本次請求的 URL 許可權，如果有就透過請求，讓他訪問資源，如果沒有就返回，提示使用者不具有該資源的訪問許可權。

## 按鈕級別的許可權管理

比如後臺的選單按鈕，每一個按鈕對應的是一個資源。在顯示按鈕的時候，可以透過後臺先查詢此使用者具有哪些資源的訪問許可權，然後根據對應的許可權顯示對應的按鈕。此方法可以做到按鈕級別的許可權控制。

web網頁，還可以同過自定義標籤來實現入口的控制。比如：
```xml
<diy:power value="${url}"><a>許可權按鈕</a></diy:power>
``` 
此標籤根據 value，在後臺查詢資料庫去判斷是否具有許可權，來決定標籤內容是否顯示。
