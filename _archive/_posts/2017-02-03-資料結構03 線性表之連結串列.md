1、為什麼要使用連結串列？

2、連結串列的儲存結構？

## 1、為什麼要使用連結串列

透過上一篇的學習，我們知道順序表存在一些問題，主要有以下兩個方面。

1、順序表的長度是固定的，如果超出分配的長度就會造成溢位，如果存放的資料太少就會造成空間浪費。 

2、在插入元素和刪除元素時（尤其插入和刪除的位置不在尾部時），會移動大量的元素，造成效能和效率低下。

基於以上問題，使用連結串列可以很好地避免順序表中出現的問題。這也是我們要使用連結串列的原因。

## 2、連結串列的儲存結構

![][1]

1、從上圖可以看出，單連結串列中的每個節點都包含一個“資料域”和一個“指標域”。“資料域”中包含當前節點的資料，“指標域”中包含下一節點的儲存地址。

2、頭指標 head 是指向開始節點的，結束節點沒有後繼節點，所以結束節點的指標域為空，即 null。

3、**連結串列在物理儲存結構上不連續，邏輯上連續；大小不固定。**

4、根據連結串列的構造方式的不同可以分為：

*   **單向連結串列**
*   **單向迴圈連結串列**
*   **雙向連結串列**
*   **雙向迴圈連結串列**

### 2-1、單向連結串列

連結串列的**每個節點中只包含一個指標域**，叫做**單向連結串列**（即構成連結串列的每個節點只有一個指向後繼節點的指標）

### 2-2、單向迴圈連結串列

單向迴圈連結串列和上面講的單向連結串列有點相似，都是透過節點的指標指向它的下一個節點，然後這樣連線起來，但不同的地方是單向連結串列的最後一個節點的指標為null，而單向迴圈連結串列的最後一個節點的指標指向的是頭節點，這樣構成了一個迴圈的節點的環。下面是單向迴圈連結串列的示意圖：

![][3]

### 2-3、**雙向連結串列**

聽名字可能就能猜到雙向連結串列就是連結串列的節點包含兩個指標，一個指標是指向它的下一個節點，另一個指標指向它的上一個節點。這樣雙向連結串列就可以透過第一個節點找到最後一個節點，也可以透過最後一個節點找到第一個節點，雙向連結串列的示意圖：

## 3、連結串列的常用操作

**連結串列常用的操作有：**（以單向連結串列為例）

### 3-1、插入節點

思路：需要迴圈查詢此節點應該插入的位置。所以時間複雜度為O(n)

示意圖：

![][6]

### 3-2、刪除節點

思路：迴圈查詢要刪除的節點的位置，所以時間複雜度為O(n)

示意圖：

![][7]

### 3-3、查詢節點

思路：與插入節點和刪除節點的方法類似，需要遍歷連結串列中的節點，所以時間複雜度為O(n)

### 3-4、獲取連結串列的長度

思路：不像順序表那樣連續儲存，獲取表的長度非常容易；在連結串列中，資料不是連續儲存的，因此需要迴圈遍歷才能求得連結串列的長度，所以時間複雜度為O(n) 

## 4、連結串列的常用操作的程式碼實現

### 4-1、插入節點

在程式碼裡面用到了一個變數 position，它的含義如下圖所示：

![][8]

注意：

1、頭節點不儲存資料，它的資料域為null，它的地址域儲存了第1個節點的地址

2、頭節點不算進連結串列的長度，position 從頭節點後面的節點開始算起

3、**每個節點裡面分別有資料域和地址域**  

下面是具體的實現程式碼：

先建立一個節點類： Node.java

<pre>package com.demo;

/**
 * 節點類
 */
public class Node {
    Object element; // 資料域
    Node next; // 地址域

    // 節點的構造方法
    public Node(Object element, Node next) {
        this.element = element;
        this.next = next;
    }

    // Gettet and Setter
    public Node getNext() {
        return this.next;
    }

    public void setNext(Node next) {
        this.next = next;
    }

    public Object getElement() {
        return this.element;
    }

    public void setElement(Object element) {
        this.element = element;
    }

}</pre>

**注意每個節點裡面分別有資料域和地址域！** 

定義連結串列類：MyLinkedList.java

<pre>package com.demo;

/**
 * 自己定義的一個連結串列類
 */
public class MyLinkedList {

    // 頭節點
    private Node headNode;
    // 用來遍歷連結串列的臨時節點
    private Node tempNode;

    // 連結串列的初始化方法
    public MyLinkedList() {
        // 初始化時，連結串列裡面只有1個節點，所以這個節點的地址域為null
        Node node = new Node("王重陽", null);
        // 頭節點不儲存資料，它的資料域為null，它的地址域儲存了第1個節點的地址
        headNode = new Node(null, node);
    }

    /**
     * 1、插入節點：時間複雜度為O(n)
     * @param newNode 需要插入的新節點
     * @param position 這個變數的範圍可以從0到連結串列的長度，注意不要越界。
     *                 頭節點不算進連結串列的長度，
     *                 所以從頭節點後面的節點開始算起，position為0
     */
    public void insert(Node newNode, int position) {
        // 透過position變數，讓tempNode節點從頭節點開始遍歷，移動到要插入位置的前一個位置
        tempNode = headNode;
        int i = 0;
        while (i < position) {
            tempNode = tempNode.next;
            i++;
        }
        newNode.next = tempNode.next;
        tempNode.next = newNode;
    }

    // 遍歷連結串列，列印出所有節點的方法
    public void showAll() {
        tempNode = headNode.next;
        while (tempNode.next != null) {
            System.out.println(tempNode.element);
            tempNode = tempNode.next;
        }
        System.out.println(tempNode.element);
    }
}</pre>

測試類：TestMyLinkedList.java

<pre>package com.demo;

public class TestMyLinkedList {
    public static void main(String[] args) {
        MyLinkedList myLinkedList = new MyLinkedList();

        Node newNode1 = new Node("歐陽鋒", null);
        Node newNode2 = new Node("黃藥師", null);
        Node newNode3 = new Node("洪七公", null);

        myLinkedList.insert(newNode1, 0);
        myLinkedList.insert(newNode2, 0);
        myLinkedList.insert(newNode3, 0);

        myLinkedList.showAll();
    }
}</pre>

### 4-2、刪除節點

在 MyLinkedList.java 中新增刪除節點的方法

<pre>    /**
     * 2、刪除節點：時間複雜度為O(n)
     * @param position
     */
    public void delete(int position) {
        // 這裡同樣需要用tempNode從頭開始向後查詢position
        tempNode = headNode;
        int i = 0;
        while (i < position) {
            tempNode = tempNode.next;
            i++;
        }
        tempNode.next = tempNode.next.next;
    }</pre>

    public Node find(int position) {

### 4-4、獲取連結串列的長度

在 MyLinkedList.java 中新增方法

<pre>    /**
     * 4、獲取連結串列的長度：時間複雜度為O(n)
     * @return
     */
    public int size() {
        tempNode = headNode.next;
        int size = 0;
        while (tempNode.next != null) {
            size = size + 1;
            tempNode = tempNode.next;
        }
        size = size + 1; // tempNode的地址域為null時，size記得加上最後一個節點
        return size;
    }</pre>

## 5、總結

1、連結串列**插入和刪除**一個元素的時間複雜度均為O(n)，另外，連結串列**讀取**一個資料元素的時間複雜度也為O(n) 

2、**順序表和單連結串列的比較：**

**①順序表：**

　　**優點**：主要優點是**讀取元素的速度較快**，以及記憶體空間利用效率高；

　　**缺點**：主要**缺點是需要預先給出順序表的最大元素個數**，而這通常很難準確做到。當實際的元素個數超過了預先給出的個數，會發生異常。另外，**順序表插入和刪除操作時需要移動較多的資料元素**。

**②單連結串列：**

　　**優點**：主要優點是**不需要預先給出最大元素個數**。另外，**單連結串列插入和刪除操作時不需要移動資料元素**；

　　**缺點**：主要缺點是**每個節點都需要分為地址域和資料域，因此單連結串列的空間利用率略低於順序表**。另外，單連結串列讀取一個元素的時間複雜度為O(n)  ；而順序表讀取一個元素的時間複雜度為O(1)
