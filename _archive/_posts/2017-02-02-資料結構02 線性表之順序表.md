## 1、線性表是什麼 

（1）線性表是最基本、最簡單的一種資料結構。

（2）線性表中元素之間的關係是一對一的關係，即除了第一個和最後一個資料元素之外，其它資料元素都是首尾相接的。

（3）如果是一對多就用樹來表示，如果多對多就用網狀來表示。

（4）線性表具有以下幾個特徵：

　　①有且只有一個“首”元素
  
　　②有且只有一個“尾”元素
  
　　③除“首”元素之外，其餘元素都有唯一的前驅元素。
  
　　④除“尾”元素之外，其餘元素都有唯一的後繼元素。  

## 2、線性表的兩種儲存結構

（1）順序表，即線性表用順序儲存結構儲存資料，資料是連續的。這一篇文章總結的就是順序表 
（2）連結串列，即線性表用鏈式儲存結構儲存資料，資料是不連續的。

## 4、順序表的常見操作和程式碼實現 

順序表主要有以下幾個常見操作，我們一般用陣列來儲存資料

**1、初始化**

思路：將陣列的長度 size 設為0，時間複雜度為O(1)。

**2、求順序表的長度**

思路：獲取陣列的 size 值，時間複雜度為O(1)。

**3、插入元素**

思路：分兩種情況，一種是插入位置在陣列的末尾，這種情況的時間複雜度為O(1) 。另一種情況是插入位置在陣列的頭部，這時候被插入元素的後續元素都要依次向後移動一位，也就是說整個陣列都會移動，所以最壞時間複雜度為O(n)。

**4、刪除元素**

思路：同樣分兩種情況，一種是刪除位置在陣列的末尾，不用移動任何元素，因此時間複雜度為O(1)，另一種情況是刪除位置在陣列的頭部，這時候被刪除元素的後續元素都要依次向前移動一位，因此時間複雜度為O(n)。

**5、按序號查詢元素**

思路：因為順序表的儲存地址是連續的，所以第n個元素的地址偏移量公式為：(n-1)*單元儲存長度，不用移動任何元素，因此時間複雜度為O(1)。

程式碼實現：

<pre>package com.demo;

public class SequenceList {
    // 預設的長度
    final int defaultSize = 10;
    // 最大長度
    int maxSize;
    // 當前長度
    int size;
    // 順序表物件
    Object[] list;

    /**
     * 1、順序表的初始化方法
     * 時間複雜度為O(1)
     *
     * @param size
     */
    private void init(int size) {
        this.size = 0;
        maxSize = size;
        list = new Object[size];
    }

    /**
     * 2、求順序表的長度
     * 時間複雜度為O(1)
     *
     * @return
     */
    public int size() {
        return size;
    }

    /**
     * 3、插入元素
     * 時間複雜度為O(n)
     *
     * @param index
     * @param obj
     * @throws Exception
     */
    public void insert(int index, Object obj) throws Exception {
        // 如果當前順序表已滿，那就不允許插入資料
        if (size == maxSize) {
            throw new Exception("順序表已滿，無法插入！");
        }
        // 插入元素的位置編號是否合法
        if (index < 0 || index > size) {
            throw new Exception("插入元素的位置編號不合法！");
        }
        // 移動元素。 要從後往前操作
        for (int i = size - 1; i >= index; i--) {
            list[i + 1] = list[i];
        }
        list[index] = obj;
        size++;
    }

    /**
     * 4、刪除元素
     * 時間複雜度為O(n)
     *
     * @param index
     * @throws Exception
     */
    public void delete(int index) throws Exception {
        // 判斷當前順序表是否為空
        if (size == 0) {
            throw new Exception("順序表為空，無法刪除！");
        }
        // 刪除元素的位置編號是否合法
        if (index < 0 || index > size - 1) {
            throw new Exception("刪除元素的位置編號不合法！");
        }
        // 移動元素。 要從前往後操作
        for (int i = index; i < size - 1; i++) {
            list[i] = list[i + 1];
        }
        size--;
    }

    /**
     * 5、按序號查詢元素
     * 時間複雜度為O(1)
     *
     * @param index
     * @return
     * @throws Exception
     */
    public Object get(int index) throws Exception {
        // 查詢元素的位置編號是否合法
        if (index < 0 || index > size - 1) {
            throw new Exception("查詢元素的位置編號不合法！");
        }
        return list[index];
    }

}</pre>

**注意：**

**插入元素時，移動元素要從後往前操作，否則元素會被覆蓋**。

**刪除元素時，移動元素要從前往後操作。**

## **5、總結**

順序表插入元素和刪除元素的時間複雜度為O(n)

順序表查詢一個元素的時間複雜度為O(1) ，因為順序表可以透過下標直接訪問，所以時間複雜度是固定的，和問題規模無關。

順序表的優點：隨機訪問的速度很快；空間利用率高（儲存空間連續分配，不存在浪費）。

順序表的缺點：大小固定（一開始就要固定順序表的最大長度）；插入和刪除元素需要移動大量的資料。
